# Frontend Files — Role & Explanation

This document explains the purpose and role of every frontend file in the `client/` directory.

---

## Core Files

### `index.html`
The single HTML entry point for the React app. Contains:
- SEO meta tags (title, description, keywords, author, theme-color)
- The `<div id="root">` where React mounts the entire app
- Script tag loading `main.jsx`

### `src/main.jsx`
The JavaScript entry point. It renders the root `<App />` component into the `#root` DOM element using `ReactDOM.createRoot`. This file is auto-generated by Vite and rarely needs editing.

### `src/App.jsx`
**The main orchestrator of the entire portfolio.** It:
- Fetches all data (profile, services, projects, skills) from the Express backend API on mount using `useEffect` + `fetch`
- Includes fallback data so the frontend works even if the backend is offline
- Renders the full page layout: `Sidebar` → `Main Content` (Hero, Services, Projects, Skills, Contact, Footer) → `NavDots`
- Shows a loading spinner while data is being fetched

### `src/App.css`
Styles for the loading/splash screen (spinner animation) shown while the app fetches data from the API.

### `src/index.css`
**The global design system.** This is the most important CSS file — it defines:
- **CSS Variables (tokens):** All colors, gradients, shadows, border-radius, fonts, transitions used across the app
- **Reset:** Box-sizing, margin/padding reset, scrollbar styling
- **Layout:** `.app-layout`, `.main-content` — the sidebar + content + nav 3-column structure
- **Section utilities:** `.section`, `.section-label`, `.section-title`, `.gradient-text` — reusable styles for every page section
- **Card system:** `.glass-card` — the glassmorphism card with hover glow effect
- **Buttons:** `.btn-gradient` and `.btn-outline` — gradient and outline button styles
- **Animations:** `fadeInUp`, `fadeInLeft`, `floatAnimation`, `skillBarFill`, etc.
- **Background effects:** `.bg-glow` — the soft colored blurs in the background
- **Responsive breakpoints:** Media queries for tablet (992px) and mobile (768px)

## Components

Each component has a `.jsx` file (logic + markup) and a matching `.css` file (styles).

### `components/Sidebar.jsx` + `Sidebar.css`
**The fixed left sidebar** visible on every section. Contains:
- Profile photo inside an animated gradient ring
- Green "available" status dot
- Name (gradient text) and title
- Social media links (LinkedIn, GitHub, X/Twitter, Email) as icon buttons
- "Download CV" gradient button
- Stats cards at the bottom: Experience (3+ yrs), Projects (15+), Clients (10+)

On mobile, it collapses into a horizontal bar at the top of the page.

### `components/NavDots.jsx` + `NavDots.css`
**The fixed right-side navigation rail** with icon buttons for each section (Home, Services, Projects, Skills, Contact). Features:
- Uses **Intersection Observer** to automatically highlight the active section as you scroll
- Clicking a dot smoothly scrolls to that section
- Shows tooltip labels on hover
- Active dot gets a cyan border + left accent bar

On mobile, it becomes a fixed bottom navigation bar.

### `components/Hero.jsx` + `Hero.css`
**The landing/intro section** — the first thing visitors see. Features:
- Greeting with animated waving hand emoji 👋
- Large name heading
- **Typing animation** that cycles through titles: "Frontend Development" → "Backend Systems" → "Full Stack Solutions" → "Creative Design" with a blinking cursor
- Bio paragraph
- Two CTA buttons: "View My Work" (gradient) and "Let's Talk →" (outline)
- Decorative floating gradient circles in the background
- "Scroll Down" indicator with animated line at the bottom

### `components/Services.jsx` + `Services.css`
**The "What I Do" section** showing 3 specialty cards in a responsive grid:
1. Responsive Development
2. Digital Experience
3. People Management

Each card is a `.glass-card` with:
- A faint number watermark (01, 02, 03)
- An AI-generated illustration
- Title (gradient text) and description
- Hover effect: lifts up, glows, shows top gradient border

### `components/Projects.jsx` + `Projects.css`
**The portfolio section** displaying 4 project cards in a 2-column grid. Each card shows:
- Folder icon (cyan, with gradient background)
- External link icon (top-right corner)
- Project title and description
- Tech stack tags as small pills (cyan border + background)

Projects shown: E-Commerce Platform, Task Management App, AI Chat Interface, Fitness Tracker.

### `components/Skills.jsx` + `Skills.css`
**The expertise section** with a 2-column layout:
- **Left column:** 8 skill bars with animated gradient fills (triggered when scrolled into view via Intersection Observer). Each bar shows the skill name and percentage, and the fill animates from 0% to the target width with staggered delays. A glowing cyan dot appears at the end of each bar.
- **Right column:** A glass card with tech badge pills (React, Next.js, JavaScript, TypeScript, Node.js, Express, MongoDB, PostgreSQL, Git, Docker, Figma, VS Code, REST APIs, GraphQL). Badges have hover effects.

### `components/Contact.jsx` + `Contact.css`
**The contact section** with a 2-column layout:
- **Left column:** Contact info — heading, description paragraph, and 3 detail items (Email, Phone, Location) each with an icon box
- **Right column:** A glass card containing a **contact form** with Name, Email, and Message fields. On submit, it sends a POST request to `http://localhost:5000/api/contact`. Shows success/error messages below the submit button.

### `components/Footer.jsx` + `Footer.css`
**The page footer** at the very bottom. Shows:
- Copyright text: "© 2001 - {current year}. All rights reserved."
- "Built with React & Node.js" with gradient-highlighted tech names
- A subtle gradient accent line at the top border
